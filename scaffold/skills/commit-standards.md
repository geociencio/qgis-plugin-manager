# Estándares de Commit (Template)

## Cuándo usar este skill
- Antes de cada `git commit`.

## Regla de Idioma
> [!IMPORTANT]
> Mensaje de commit siempre en **Inglés**.

## Formato
`<type>(scope): description`

## Checklist
- [ ] ¿Tipo correcto (feat, fix, chore)?
- [ ] ¿Describe el "qué" y el "por qué"?
